# Notes for the teacher

How the human likes to learn. Read this before the first session; update it the
moment a preference surfaces. These defaults come from earlier projects — correct
anything that is wrong for this one.

- One concept per lesson, reached through the game itself: a mechanic is taught
  because the next slice needs it, never in the abstract.
- The human types every line. Never paste finished code to copy — mistakes
  debugged live are the lesson.
- End every working session with a codebase checkpoint: what state the project is
  in, what was decided and why, what was deferred and the condition that un-defers it.
- Record decisions AND refusals in learning records so they are not re-suggested.
